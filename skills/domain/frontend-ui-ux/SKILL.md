# Frontend UI/UX

Use this skill for UI, styling, interaction, accessibility, responsive layout, and visual polish tasks.

## Rules
- Preserve the existing design system, component patterns, and visual language when present.
- Avoid generic AI-looking layouts, predictable cards, and default purple gradients.
- Make desktop and mobile states work.
- Prefer semantic HTML and accessible interaction states.
- Do not add animation or visual complexity that conflicts with product usability.

## Process
1. Identify existing components, styles, tokens, and layout conventions.
2. Define the page or component's purpose and tone.
3. Implement the smallest visual change that achieves the goal.
4. Check responsive behavior and interaction states.
5. Run available lint/build checks when practical.

## Output
- Design direction
- Files changed
- Responsive/accessibility considerations
- Verification performed
